YAML Learning Material Concepts
===============================
- Yaml is a data serialization language.
- Yaml stands for Yaml ain't a markup language.
- It's file extension are like - .yaml, .yml
- It is superset of JSOn: any valid json file is a also a valid YAML file
- Its a case sensitive language

Site For YAML Language
----------------------
Official Site: [https://yaml.org/]
Yaml Tools: [https://onlineyamltools.com/]



Difference With Other Serialization Language
--------------------------------------------
See the different serialization languages:
   1. YAML Version
      microservices:
         - app: user-authentication
           port: 9000
           version: 1.0

   2. XML Version:
      <microservices>
         <microservice>
            <app>user-authentication</app>
            <port>9000</app>
            <version>1.0</version>
         </microservice>
      </microservices>

   3. JSON Version:
      {
         microservices:[
            {
               app: "user_authentication",
               portL: 9000,
               version: "1.0"
            }
         ]
      }

Booleans In Yaml
----------------
- Yaml three types of expression to provide boolean values
   1. true/false
   2. yes/no
   3. on/off


Multiline Or Single Line In Yaml
--------------------------------
a. Multiline String
   - use | to setup the multiline string
   - example:
      multilineString: |
         this is a multiline string
         which can be used to setup 
         a multiline string having multiline

b. SingleLine String
   - use > to setup the single line string
   - example:
      singlelineString: >
         this is a single line string 
         whcih later can be single line
         because of the special character arrow(>)


Placeholders In Yaml
--------------------
- placeholders are used in Yaml
- to use placeholder {{ }} is used

   data: {{ .Values.series.app }}

Components In Yaml
------------------
- component in Yaml is used to configure or setup multiple document inside a yaml file
- its basically separate the document in a yaml file
- syntax of component:  
      --- 
         appVersion: v1
         data: 
            appName: 'Visor'
            version: 1.1.2
